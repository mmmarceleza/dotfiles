# Global Guidelines

## Language
- Always respond in United States English
- Write all code comments in English
- Use clear, concise communication

## Code Style
- Prioritize readability over cleverness
- Follow existing project standards when available
- Use descriptive variable and function names
- Keep functions small and single-purpose
- Prefer explicit over implicit

## Error Handling
- Always handle errors explicitly, never silently ignore them
- Use meaningful error messages
- Log errors with sufficient context for debugging

## Security
- Never expose credentials in code
- Use environment variables for sensitive configurations
- Validate and sanitize all user inputs
- Follow the principle of least privilege

## Testing
- Write tests for new functionality
- Prefer unit tests for business logic
- Use descriptive test names that explain the expected behavior

## Documentation
- Document public APIs and complex logic
- Keep documentation close to the code
- Update docs when changing related code

## Git
- Use conventional commit format (type: subject + optional body)
- No emoji
- Use imperative mood (e.g., "Add feature" not "Added feature")
- Reference issues and breaking changes in the body
- Keep commits atomic: one logical change per commit
- No 'Generated with Claude Code' or 'Co-Authored-By' lines

## Workflow Preferences
- Ask before making large refactors
- Explain reasoning for non-obvious decisions
- When uncertain, present options rather than assuming
